# Веб-интерфейс (Next.js + FastAPI)

UI для управления нарративным пайплайном: задать тему, **вручную править
артефакты на любом этапе**, следить за прогрессом агентов в реальном времени.

```
┌─────────────┐   HTTP + SSE   ┌──────────────┐   LangGraph   ┌──────────┐
│ Next.js UI  │ ◀────────────▶ │ FastAPI мост │ ◀───────────▶ │  Пайплайн │
│ (web/)      │   :3000→:8000  │ (server/)    │  + SQLite     │ (8 ботов) │
└─────────────┘                └──────────────┘  checkpointer └──────────┘
```

## Что умеет UI
- **Тема/жанр/язык/адалт-главы** — форма запуска.
- **Прогресс агентов** — live-лог через SSE + полоса стадий (готово/активно).
- **Ручные правки на всех этапах** — каждый артефакт (логлайн, синопсис,
  карточки, план/диалоги/адалт/перевод каждой главы) редактируется и
  сохраняется в общую память (`update_state` чекпоинта).
- **Пошаговый режим** — пауза после каждого бота; правишь результат и жмёшь
  «Продолжить». Под капотом — LangGraph `interrupt_after`.
- **Отчёты редактора** (Бот 7) показываются под каждой главой с severity.

## Запуск

### 1. Бэкенд (FastAPI)
```bash
cd /home/user/ai-narrative
pip install -e ".[server]"        # или: .venv/bin/pip install -e ".[server]"
# .env уже настроен (OpenRouter ключ, модели)
uvicorn server.app:app --reload --port 8000
```

### 2. Фронтенд (Next.js)
```bash
cd web
npm install
cp .env.local.example .env.local   # если бэкенд не на localhost:8000
npm run dev                        # http://localhost:3000
```

Открой http://localhost:3000 → задай тему → «Запустить пайплайн».

## API (для справки)
| Метод | Путь | Назначение |
|---|---|---|
| POST | `/api/runs` | старт прогона (theme/genre/lang/adult_chapters/step_mode) → thread_id |
| GET | `/api/runs/{id}/events` | SSE: log/state/status события |
| GET | `/api/runs/{id}/state` | текущий state + status + next |
| PATCH | `/api/runs/{id}/state` | ручная правка артефактов (`{patch:{...}}`) |
| POST | `/api/runs/{id}/resume` | продолжить с паузы/после правок |

## Заметки
- **Один процесс — общий SQLite-checkpointer** (`narrative_state.db`). Thread_id
  = одна новелла; переживает рестарт сервера (resume по тому же id).
- Step-mode компилирует граф с `interrupt_after` по всем стадиям. Авто-режим —
  без пауз, правки доступны после завершения.
- Прод-хардинг (не сделано, dev-инструмент): аутентификация, на запись в SQLite
  из нескольких потоков стоит вынести очередь; CORS сейчас `*`.
