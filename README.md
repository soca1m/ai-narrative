# ai-narrative — конвейер ботов для визуальных новелл 18+

LangGraph-пайплайн из 8 ботов (+ редактор) по нарративному проекту. Результат
каждого бота передаётся следующему через **общую память** (LangGraph state +
checkpointer). Редактор (Бот 7) ловит ошибки/недостоверность и **гонит правку
обратно боту-виновнику** с лимитом попыток.

## Конвейер

```
Бот1 Логлайн → Бот2 Синопсис → Бот3 Персонажи → Бот4 Структура
        │
        └─ поглавный цикл (одна глава за раз):
             Бот5 Диалоги → Бот6 Адалт* → Бот7 Редактор ──┐
                  ▲                                         │
                  └────── правка (responsible_node) ◄───────┘
                          critical-замечание + попытки < MAX_REVISIONS
        │
        └─ Бот8 Перевод → END

* Бот6 запускается только на «адалтных точках» (Chapter.is_adult_point),
  которые нарративщик проставляет вручную — как в исходном доке.
```

### Кто чем рулит
| Бот | Узел | Модель |
|---|---|---|
| 1 Логлайн | `logline` | Claude / OpenRouter |
| 2 Синопсис | `synopsis` | Claude / OpenRouter |
| 3 Персонажи | `characters` | Claude / OpenRouter |
| 4 Структура | `structure` | Claude / OpenRouter |
| 5 Диалоги | `dialogue` | Claude / OpenRouter |
| 6 Адалт | `adult` | **модель без цензуры** (Magnum v4 72B / локальный) |
| 7 Редактор | `editor` | Claude / OpenRouter (structured JSON) |
| 8 Перевод | `translation` | Claude / OpenRouter |

## Маршрутизация правок (линчпин)

Редактор возвращает **структурированные** `Finding` (а не markdown), поэтому
условное ребро может ветвиться машинно. Каждый `Finding` несёт
`responsible_node` — кому переделывать:
- проблема в тексте главы (имя/род/стиль/реплика) → `dialogue`
- проблема в откровенной сцене → `adult`
- неверно заложена мотивация в карточке → `characters`
- дыра в плане/арке → `structure`

Правка **точечная** (как в памятке: «не начинай заново — переделай пункт 3»):
боту передаётся прежний черновик + конкретный фидбек. Цикл ограничен
`MAX_REVISIONS`; после исчерпания глава уходит дальше, а замечания остаются в
`editor_reports` для ручного разбора человеком.

## Общая память
- `State` (`src/narrative/state.py`) — накопительные артефакты всех ботов.
- Checkpointer (dev: `MemorySaver`, прод: SQLite/Postgres) хранит state по
  `thread_id`. **Один thread_id = одна новелла** (= «новый чат на проект»).

## Запуск
```bash
pip install -e .
cp .env.example .env   # вписать ключи
python scripts/run.py --theme "Летний теннисный клуб, власть, мистика" \
    --genre "комедийная драма" --lang English --thread korona-001
```
